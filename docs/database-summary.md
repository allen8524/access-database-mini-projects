# Database Summary

이 문서는 `databases` 폴더에 포함된 Microsoft Access 데이터베이스 파일의 주요 테이블 구성과 설계 목적을 정리한 문서입니다.

## 1. 주소록 관리 데이터베이스

파일명: `databases/address-book.accdb`

주소록과 가족 정보를 관리하기 위한 Microsoft Access 데이터베이스입니다. 연락처 기본 정보와 가족 정보를 분리하여 관리할 수 있도록 구성했습니다.

### 주요 테이블

| 테이블 | 설명 |
| --- | --- |
| 주소록 | 이름, 전화번호, 생일, 주소 등 연락처 기본 정보 관리 |
| 가족 | 주소록 데이터와 연결되는 가족 구성원 정보 관리 |

### 주소록 테이블

| 컬럼 | 설명 |
| --- | --- |
| ID | 주소록 데이터의 기본 식별자 |
| 음력 | 생일의 음력 여부 |
| 생일 | 연락처 생일 |
| 이름 | 연락처 이름 |
| 전화 | 연락처 전화번호 |
| 우편번호 | 주소의 우편번호 |
| 주소 | 상세 주소 |
| 사진 | 연락처 사진 정보 |

### 가족 테이블

| 컬럼 | 설명 |
| --- | --- |
| ID | 가족 데이터의 기본 식별자 |
| 주소록_ID | 주소록 테이블과 연결되는 외래키 |
| 생일 | 가족 구성원 생일 |
| 관계 | 주소록 대상자와의 관계 |
| 이름 | 가족 구성원 이름 |
| 학력 | 가족 구성원 학력 |
| 직업 | 가족 구성원 직업 |
| 비고 | 추가 메모 |

### 테이블 관계

- `가족.주소록_ID`는 `주소록.ID`를 참조합니다.
- 주소록 1건에 여러 가족 정보를 연결할 수 있는 구조입니다.

### 설계 목적

- 연락처 정보를 한 곳에서 관리
- 연락처별 가족 정보를 별도 테이블로 분리
- 주소록과 가족 정보를 관계형 구조로 연결
- 샘플 데이터를 활용해 주소록 관리 흐름 구성

## 2. 판매관리 데이터베이스

파일명: `databases/sales-management.accdb`

제품 분류, 제품 정보, 판매 내역, 판매 상세 내역을 관리하기 위한 Microsoft Access 데이터베이스입니다. 판매 1건에 여러 판매 상세 내역이 연결되는 구조로 구성했습니다.

### 주요 테이블

| 테이블 | 설명 |
| --- | --- |
| 종류 | 제품 분류 정보 관리 |
| 제품 | 제품명, 단가, 종류번호 관리 |
| 판매 | 판매일, 판매시간, 합계, 결제상태 관리 |
| 판매내용 | 판매번호별 제품, 수량, 금액 관리 |

### 종류 테이블

| 컬럼 | 설명 |
| --- | --- |
| 종류번호 | 제품 종류의 기본 식별자 |
| 종류명 | 제품 분류명 |

### 제품 테이블

| 컬럼 | 설명 |
| --- | --- |
| 제품번호 | 제품의 기본 식별자 |
| 종류번호 | 제품 종류와 연결되는 외래키 |
| 제품명 | 제품 이름 |
| 단가 | 제품 단가 |

### 판매 테이블

| 컬럼 | 설명 |
| --- | --- |
| 판매번호 | 판매 내역의 기본 식별자 |
| 날짜 | 판매 날짜 |
| 시간 | 판매 시간 |
| 합계 | 판매 총합계 |
| 결제상태 | 현금, 카드, 대기 등 결제 상태 |

### 판매내용 테이블

| 컬럼 | 설명 |
| --- | --- |
| 번호 | 판매 상세 내역의 기본 식별자 |
| 판매번호 | 판매 테이블과 연결되는 외래키 |
| 제품번호 | 제품 테이블과 연결되는 외래키 |
| 단가 | 판매 당시 제품 단가 |
| 수량 | 판매 수량 |
| 금액 | 단가와 수량을 기준으로 계산되는 금액 |

### 테이블 관계

- `제품.종류번호`는 `종류.종류번호`를 참조합니다.
- `판매내용.판매번호`는 `판매.판매번호`를 참조합니다.
- `판매내용.제품번호`는 `제품.제품번호`를 참조합니다.
- 판매 1건에 여러 판매내용이 연결될 수 있는 구조입니다.

### 설계 목적

- 제품 분류와 제품 정보를 분리 관리
- 판매 기본 정보와 판매 상세 정보를 분리 관리
- 판매번호를 기준으로 판매 내역과 판매내용을 관계형으로 구성
- 판매 상세 데이터를 통해 제품별 판매 수량과 금액을 관리

## Git 관리 주의사항

- `.accdb` 파일은 바이너리 파일이므로 GitHub에서 코드처럼 변경 내용을 비교하기 어렵습니다.
- Access에서 수정한 뒤에는 파일 전체를 다시 커밋해야 합니다.
- Access 임시파일인 `.laccdb`는 커밋하지 않습니다.
- 내부 확인은 Microsoft Access 설치 환경에서 진행하는 것이 가장 정확합니다.
